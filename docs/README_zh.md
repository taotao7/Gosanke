## 这是什么？

Gosanke 会把 **Claude**、**ChatGPT** 和 **Gemini** 分别打开在三个独立的弹出窗口中，另有一个浮动的**中控窗口**负责统一输入。由于这三个站点都禁止 `iframe` 嵌入，多窗口是唯一可行的方案。

## 功能一览

### 三站同发
- 在中控输入框输入一次，同时发送到三个 AI 站点
- 支持附带图片，文本和图片一起分发到每个站点
- 支持直接在中控输入框粘贴截图（Ctrl/Cmd+V）

### 智能图片上传
- 优先使用合成粘贴事件注入（模拟原生粘贴行为，兼容性更好）
- 自动降级到 file input 注入（当粘贴方式不被支持时）
- 每个站点独立的上传策略，自动适配不同平台的 DOM 结构

### 单条消息转发
- 鼠标悬浮在 Claude / ChatGPT / Gemini 的任意一条消息上，会出现转发按钮
- 一键将该条消息转发到另外两个平台中的任意一个
- 实时状态反馈：发送中 → 已发送 / 失败

### 自适应布局
- 根据屏幕分辨率自动调整窗口大小
- 小屏幕使用更紧凑的间距，最大化站点窗口面积
- 大屏幕保留适度留白，控制器尺寸按比例缩放
- 上二下一布局，中控浮窗居中
- 在中控中拖拽卡片即可调整站点窗口的摆放顺序
- 手动点击「重新排版」可以重置为模板布局
- 窗口位置会自动保存，下次启动自动恢复

### 状态检测
- 实时检测各站点的登录状态、输入框可用性、是否有回复输出
- 状态广播到扩展 popup 启动器和中控工作台
- 使用心跳 + MutationObserver 确保状态准确可靠

### 国际化
- 支持英语和中文，根据浏览器语言自动切换
- 所有界面文字、状态提示、错误信息均已本地化

## 架构

| 窗口 | 职责 |
|------|------|
| **Popup** | 工具栏启动器 — 展示状态概览，打开工作台 |
| **Controller** | 浮动中控 — 统一输入、拖拽换位、图片附件 |
| **Claude / ChatGPT / Gemini** | 三个独立弹出窗口，注入内容脚本 |

所有通信通过后台 service worker 的 `browser.runtime.sendMessage` 完成。

## 开发

```bash
bun install
bun run dev
```

## 构建

```bash
bun run build
```

构建产物在 `.output/chrome-mv3/` 目录下。在 Chrome 扩展管理页面选择「加载已解压的扩展程序」即可测试。

## 权限说明

| 权限 | 用途 |
|------|------|
| `storage` | 保存窗口布局、摆放顺序，跨会话恢复 |
| `tabs` | 创建、查询和管理站点弹出窗口 |
| `system.display` | 读取显示器工作区信息，用于自适应布局 |
| `clipboardWrite` | 发送前将图片写入剪切板 |
| 主机权限 | 向 Claude、ChatGPT、Gemini 注入内容脚本 |
